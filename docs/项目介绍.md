## 1. 项目简介

**TestForge** 是一个 Web 应用程序，它能根据你写的需求文档（比如"用户注册功能需要..."这样的文字描述），**自动生成结构化的测试用例**。

简单说：你写需求 → AI 理解需求 → 自动生成测试用例表格 → 你可以编辑、保存、导出、分享。

项目使用 AI 大模型（DeepSeek 或 Claude）来理解和分析需求文档，生成的测试用例包含用例编号、标题、模块、前置条件、测试步骤、预期结果、优先级等字段。

## 2. 功能总览与使用方法

### 2.1 首页（主页）

登录后首先看到的是主页，这里展示你的账户概览：

- **当前用户**：显示登录用户名
- **API Key 状态**：显示你是否配置了 AI 接口密钥（绿色=已配置，红色=未配置）
- **当前模型**：当前使用的 AI 模型名称
- **当前用例数**：工作区中已有的测试用例数量
- **快捷按钮**：快速跳转到"需求编辑"、"用例管理"、"数据导出"等页面

> **使用方法**：登录后自动跳转到主页。点击快捷按钮或左侧边栏菜单进入各个功能页面。

### 2.2 用户认证（登录/注册）

系统支持用户注册和登录，未登录也可以作为"游客"使用（但保存到用例库等功能需要登录）。

**注册流程**：
1. 点击登录弹窗中的"注册"标签
2. 输入用户名（字母或中文开头，只能包含字母、数字、下划线）
3. 输入密码（至少 8 位，必须包含字母和数字/特殊字符）
4. 输入确认密码
5. 完成算术验证码（防止机器注册）
6. 点击"注册"

**登录流程**：
1. 输入用户名和密码
2. 点击"登录"按钮
3. 登录成功后自动跳转到主页

**登出**：点击侧边栏底部的"退出登录"按钮。

> **小贴士**：登录后系统会提示你配置 API Key（引导设置），这是使用 AI 生成功能必需的。如果没有 API Key，可以点击"跳过"稍后配置。

### 2.3 需求编辑（核心功能）

这是整个应用的核心页面，你在这里输入需求文档，然后让 AI 生成测试用例。

**使用步骤**：
1. 在左侧的大文本框中输入或粘贴需求文档（支持纯文本和 Markdown 格式）
2. 在页面顶部的下拉菜单中选择 AI 模型（DeepSeek-Chat / Claude 等）
3. 配置 API Key（点击 API 设置按钮，输入你的 API Key）
4. 选择需要生成的测试用例字段（点击字段标签选择/取消选择）
5. 点击 **"生成用例"** 按钮
6. 等待 AI 分析需求并生成测试用例（页面会显示进度状态）
7. 生成的用例会自动显示在"用例管理"页面

**其他功能**：
- **示例载入**：点击"载入示例"快速填充一个示例需求文档
- **一键清空**：清空输入内容和生成的用例
- **生成大纲**：先生成测试大纲（按模块分组的测试想法），确认后再生成详细用例
- **文本润色**：用 AI 优化需求文档的表述
- **需求预览**：切换到"预览"标签查看 Markdown 渲染效果

> **小贴士**：需求文档写的越详细，AI 生成的用例质量越高。建议包含功能模块划分、业务流程、边界条件等。

### 2.4 用例管理

在这里查看、编辑、管理 AI 生成的所有测试用例。

**表格操作**：
- **编辑单元格**：双击表格中的任意单元格直接编辑
- **添加行**：点击"添加行"按钮在末尾新增一条用例
- **删除行**：勾选用例前的复选框，点击"删除选中"
- **复制行**：选中一条用例，点击"复制"快速复制
- **批量操作**：选中多条用例后，可以批量修改优先级、审核状态、执行结果等

**其他功能**：
- **筛选**：按优先级（P0/P1/P2/P3）、审核状态、执行结果筛选
- **排序**：点击表头可按该列排序
- **搜索**：在筛选框中输入关键词搜索用例
- **卡片视图**：切换到卡片模式浏览用例（更适合阅读）
- **标签云**：根据用例模块自动生成的标签，点击可筛选
- **撤销/重做**：支持 Ctrl+Z 撤销、Ctrl+Y 重做
- **版本历史**：查看所有历史版本，可对比差异并恢复

> **小贴士**：所有修改会自动保存到浏览器的本地存储中，关闭页面不会丢失，但建议及时导出或保存到用例库。

### 2.5 AI 生成功能详解

系统支持多种 AI 驱动的生成模式，通过**左侧的四个功能按钮**操作：

**生成用例**：AI 根据需求文档生成完整的测试用例集。这是最常用的功能，AI 会分析需求中的功能点、业务流程、边界条件等，输出结构化的测试用例表格。

**生成大纲**：让 AI 先生成一个测试大纲（按模块/功能点分组的测试思路），确认大纲方向正确后，再让 AI 根据大纲生成详细的测试用例。适合需求比较复杂、需要先确认测试范围的场景。

**文本润色**：如果需求文档写的不够清晰，可以用这个功能让 AI 帮助优化表述、统一格式、补充细节。

**生成脚本**：根据当前工作区的测试用例，自动生成 **Playwright**（一个自动化测试框架）的 Python 测试脚本，可以直接用于自动化测试。

> **小贴士**：生成过程中可以随时关闭页面，后台会继续完成生成（但不建议这样做）。如果生成失败，可以检查 API Key 是否正确配置。

### 2.6 需求追溯矩阵（RTM）

RTM（Requirements Traceability Matrix）用于追踪每个需求点是否都有对应的测试用例。

**使用方法**：
1. 切换到"RTM 追溯"页面
2. 点击"生成 RTM"按钮
3. AI 会自动分析需求文档和测试用例，建立需求与用例的对应关系
4. 系统会显示每个需求点的覆盖状态（已覆盖 / 部分覆盖 / 未覆盖）
5. 查看总体覆盖率统计

### 2.7 用例库与分享

你可以将测试用例保存到服务器端的"用例库"中，方便长期存储和团队协作。

**保存到库**：
1. 在"用例管理"页面点击"保存到库"
2. 输入用例集名称

**从库加载**：
1. 点击"从库加载"按钮
2. 浏览文件夹结构
3. 找到需要的用例集，点击"加载"

**分享用例**：
1. 在用例库中右键点击一个用例集
2. 选择"分享"
3. 输入要分享给的用户名
4. 对方会收到通知，接受后该用例集会复制到对方的用例库中

**文件夹管理**：可以创建文件夹来归类整理用例集，支持多级文件夹结构。

### 2.8 好友与协作

系统支持好友系统和团队协作。

**添加好友**：

### 3.2 前端架构

**没有构建工具**！这是一个非常"复古"但简单直接的前端架构：

- **没有** Webpack / Vite / npm 构建步骤
- **没有** React / Vue 等框架
- 使用原生 JavaScript + Bootstrap 5（从 CDN 加载）
- 使用 CSS 自定义属性（CSS Variables）实现主题切换

**页面结构**：
- `base.html`：基础模板，包含侧边栏、登录弹窗、通知等公共部分
- `index.html`：主工作区，继承 base.html，包含所有功能页面的 HTML
- `tools/sql.html` 和 `tools/bugs.html`：独立工具页面，继承 base.html

**JavaScript 模块化方式**：
虽然没用 ES Module 或 CommonJS，但通过以下方式实现了模块化：

1. 每个功能是一个独立的 `.js` 文件，放在 `modules/` 目录下
2. 使用 IIFE（立即执行函数）避免变量污染
3. 通过 `window.xxx` 在模块间共享数据（全局变量方式）
4. 通过 `initXxx()` 函数初始化各模块

**脚本加载顺序**（在 HTML 中用 `<script>` 标签控制）：

```
shared.js → auth.js → global-init.js → app.js → export.js → library.js → bugs.js → ...
```

**前端数据存储**：
- **localStorage**：需求文本、测试用例（工作区）、API Key、模型选择、主题偏好等
- **服务端 SQLite**：用例库中的用例集、Bug、生成历史、好友等

### 3.3 后端架构

**Web 框架**：FastAPI（Python 的异步 Web 框架）

**为什么用 FastAPI？**
- 自动生成 API 文档（访问 `/docs` 可看到所有接口文档）
- 原生支持异步请求（对 AI 接口调用特别有用）
- 基于 Pydantic 的请求/响应模型，自动校验数据

**三层架构**：

```
路由层 (Routes) → 接收 HTTP 请求，调用服务层
   │
服务层 (Services) → 业务逻辑处理
   │
数据层 (SQLite) → 数据持久化
```

**路由层（app/routes/）**：
每个功能模块有一个路由文件，使用 FastAPI 的 `APIRouter`：
- 处理 HTTP 请求和响应
- 参数校验
- 调用服务层的函数
- 注册在 `main.py` 中

**服务层（app/services/）**：
封装所有业务逻辑：
- 数据库操作
- AI 模型调用
- 文件导出
- WebSocket 管理等

**数据层**：
使用三个 SQLite 数据库，直接使用 `sqlite3` 模块（没有 ORM，没有 SQLAlchemy）。

### 3.4 认证机制

**不用 JWT**！系统使用一种更简单的 Token 认证方式：

1. 用户登录时，服务端生成一个随机的 48 字节 Token
2. Token 存储在服务端 SQLite 的 `sessions` 表中（有效期 30 天）
3. 前端将 Token 保存在 `localStorage` 中
4. 每次 API 请求，前端在 HTTP Header 中带上 `Authorization: Bearer <token>`
5. 服务端查表验证 Token 是否有效

**密码存储**：PBKDF2-HMAC-SHA256 算法，10 万次迭代哈希。

**游客模式**：没有 Token 的用户自动成为"游客"（user_id=0），可以使用基础功能但不能保存到服务器。

### 3.5 AI 集成

支持两种 AI 提供商：

**DeepSeek**：通过 OpenAI 兼容的 SDK 调用
**Claude**：通过 Anthropic SDK 调用

系统根据模型名称自动选择使用哪个提供商（`deepseek*` 前缀 → DeepSeek，其他 → Claude）。

**AI 调用流程**：
1. 前端发送需求文档、选择的字段、模型等参数到 `/api/generate`
2. 后端构造提示词（Prompt），包含系统指令和需求文本
3. 调用 AI 的"工具调用"（Tool Calling / Function Calling）功能
4. AI 返回结构化的 JSON 数据（测试用例数组）
5. 后端验证和解析 JSON，构建为统一的 TestCase 对象
6. 返回给前端显示

**容错机制**：
- 如果工具调用失败，自动尝试从 AI 返回的文本中提取 JSON
- 如果 API 返回 429（限流）或 5xx 错误，自动重试最多 2 次
- 如果 API 返回 401（认证失败），立即报错（说明 API Key 无效）
- 内存缓存相同参数的请求结果（1 小时过期）

### 3.6 实时通知

使用 WebSocket 实现实时推送通知：
1. 用户登录后，前端通过 `ws://host/ws?token=<token>` 建立 WebSocket 连接
2. 服务端维护一个 `用户ID → WebSocket 连接` 的映射表
3. 当有人分享用例或发送好友请求时，服务端主动推送通知给对应用户
4. 前端收到通知后，在界面右上角显示小红点提示

### 3.7 数据库设计

三个 SQLite 数据库，每个专注一个职责：

**auth.db**（用户认证）：
- `users`：用户信息（用户名、密码哈希、盐值、角色）
- `sessions`：登录会话（Token、过期时间）
- `user_settings`：用户设置（主题、模型、API Key）
- `captcha_challenges`：注册验证码

**library.db**（业务数据）：
- `folders`：用例库文件夹（支持多级树形结构）
- `test_case_sets`：测试用例集（用例以 JSON 格式存储）
- `bugs`：缺陷记录
- `generation_history`：AI 生成历史
- `shared_set_access`：用例分享记录
- `notifications`：通知消息
- `contacts`：好友关系
- `prompt_templates`：AI 提示词模板
- `specifications`：需求规格关键词

**test_data.db**（SQL 查询练习）：
- `users`、`orders`、`products`：三个演示用的数据表

### 3.8 导出功能

使用 Python 库：
- **openpyxl**：生成格式化的 Excel 文件（.xlsx）
  - 蓝色表头、优先级颜色标记、冻结首行、自动筛选
  - 中文列名，覆盖所有测试用例字段
- **CSV 模块**：生成 UTF-8 BOM 编码的 CSV 文件

## 附录：常见问题

**Q：为什么生成测试用例需要 API Key？**
A：系统调用 DeepSeek 或 Claude 的 AI 接口来生成用例，这些接口需要付费使用。你需要到对应官网注册获取 API Key。

**Q：如何获取 API Key？**
A：访问 [DeepSeek 官网](https://platform.deepseek.com/) 或 [Anthropic 官网](https://console.anthropic.com/) 注册账号并创建 API Key。

**Q：测试用例数据存在哪里？**
A：工作区中的临时数据存在浏览器的 localStorage 中（关闭页面不丢失）。保存到"用例库"后，数据存在服务器的 SQLite 数据库中。

**Q：不登录可以用吗？**
A：可以。游客模式支持输入需求、生成用例、编辑表格、导出等核心功能。但保存到用例库、查看历史、分享等功能需要登录。

**Q：支持哪些 AI 模型？**
A：DeepSeek-Chat、DeepSeek-Reasoner、Claude Sonnet 4、Claude Opus 4、Claude Haiku 4。
